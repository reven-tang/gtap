# GTAP v0.3.0 开发计划

## 🎯 目标

引入 ATR 动态止损止盈机制，替代固定百分比止损，提升回测真实性和风险控制能力。

## 📚 参考来源

- **defiplot 文章**: http://defiplot.com/blog/grid-trading-with-python/
  - 核心公式：`sl_atr = 1.5 * atr_value`
  - 止损距离：`stop_loss = entry_price ± sl_atr`
  - 止盈距离：`take_profit = entry_price ∓ (sl_atr * TPSLRatio)`，其中 `TPSLRatio = 0.5`
- **ATR 计算**: 使用 `pandas` 计算 N 日平均真实波幅（默认 N=14）

### 2. 网格引擎增强（修改 `grid.py`）

#### 2.1 配置扩展（`config.py`）

在 `GridTradingConfig` 中新增 ATR 相关参数：

```python
# ATR 动态止损止盈
use_atr_stop: bool = False          # 是否启用 ATR 止损
atr_period: int = 14                # ATR 计算周期
atr_stop_multiplier: float = 1.5    # 止损乘数（sl_atr = atr * multiplier）
atr_tp_multiplier: float = 0.5      # 止盈乘数（tp_atr = atr * multiplier）
```

#### 2.2 交易决策增强

当前 `grid_trading` 仅在网格线触发买卖。新增逻辑：

```python
# 每笔交易入场时记录 ATR 止损/止盈价位
entry_atr = atr_series.loc[timestamp]
stop_loss_price = entry_price - atr_stop_multiplier * entry_atr  # 多头持仓
take_profit_price = entry_price + atr_tp_multiplier * entry_atr
```

**触发条件检查**（在每个 K 线）：
- 价格 ≤ `stop_loss_price` → 强制平仓（止损）
- 价格 ≥ `take_profit_price` → 强制平仓（止盈）

#### 2.3 数据结构扩展

`Trade` NamedTuple 增加字段：
```python

```

## ⚠️ 风险与注意

1. **ATR 滞后性**: ATR 基于历史波动，可能无法反映突发波动
2. **参数敏感性**: `atr_stop_multiplier` 影响止损频率，需参数调优
3. **前视偏差**: ATR 必须使用当前及之前数据计算，严禁使用未来数据
4. **清仓逻辑**: 止损/止盈触发后，资产序列不应提前终止（需继续记录现金价值）
